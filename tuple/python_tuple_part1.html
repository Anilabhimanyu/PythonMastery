üìò Basic Definition

my_tuple = (10, 20, 30)

print(type(my_tuple))  # <class 'tuple'>

# Single element tuple (comma required!)

print(type(single))    # <class 'tuple'>

# Without comma - not a tuple
not_tuple = (42)
print(type(not_tuple)) # <class 'int'>
‚ö†Ô∏è Important: Single element tuples need a trailing comma: (10,) otherwise it's just a value in parentheses.

import sys

# Memory comparison
lst = [1, 2, 3, 4, 5]
tpl = (1, 2, 3, 4, 5)

print(f"List: {sys.getsizeof(lst)} bytes")   # ~120 bytes
print(f"Tuple: {sys.getsizeof(tpl)} bytes")  # ~104 bytes

# Tuples are typically 10-20% smaller!

üíº Real-World Use Cases

  • Function returns: Return multiple values elegantly
  • Dictionary keys: Multi-part keys like coordinates
  • Fixed configurations: RGB colors, coordinates, database records
  • Data integrity: Prevent accidental modifications
  • Performance critical: When speed matters
  • Named tuples: Lightweight alternative to classes
# Use case 1: Multiple return values
def get_user_info():
    return ("Alice", 25, "Engineer")

name, age, role = get_user_info()

# Use case 2: Dictionary with tuple keys
locations = {
    (40.7128, -74.0060): "New York",
    (51.5074, -0.1278): "London",
    (35.6762, 139.6503): "Tokyo"
}

# Use case 3: Named tuples
from collections import namedtuple

Point = namedtuple('Point', ['x', 'y'])
p = Point(10, 20)
print(p.x, p.y)  # 10 20

# Use case 4: Swap values
a, b = 5, 10
a, b = b, a  # Uses tuple packing/unpacking
print(a, b)  # 10 5

# Practice: Flatten nested tuple
def flatten(t):
    result = []
    for item in t:
        if isinstance(item, tuple):
            result.extend(flatten(item))
        else:
            result.append(item)
    return tuple(result)

nested = (1, (2, 3), (4, (5, 6)))
print(flatten(nested))  # (1, 2, 3, 4, 5, 6)

üìö Quick Summary

When to use Tuples:
‚úÖ Data shouldn't change (coordinates, RGB values, config)
‚úÖ Need to use as dict keys or set elements
‚úÖ Performance matters (faster than lists)
‚úÖ Want to prevent accidental modifications
‚úÖ Returning multiple values from functions
When to use Lists:
‚úÖ Need to modify data frequently
‚úÖ Need list methods (append, remove, sort, etc.)
‚úÖ Size of collection changes over time
‚úÖ Order matters and needs to be changed
