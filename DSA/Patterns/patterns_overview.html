🎓 What are DSA Patterns?

DSA Patterns are recurring problem-solving techniques that can be applied to similar types of coding problems. Instead of memorizing solutions to hundreds of problems, you learn to recognize patterns and apply the appropriate technique.

🔑 Why Learn Patterns?

  • Efficiency: Solve new problems faster by recognizing familiar patterns
  • Interview Success: Most interview problems follow known patterns
  • Deep Understanding: Learn WHY a solution works, not just HOW
  • Transferable Skills: Apply patterns across different problem domains
  • Confidence: Approach unknown problems with structured thinking

Array & String Patterns

Fundamental patterns for array and string manipulation problems

👉Two Pointers

Easy-Medium O(n)

Use two pointers moving towards or away from each other to solve problems involving pairs, triplets, or subarrays.

When to Use:

  • Finding pairs with target sum in sorted array
  • Removing duplicates from sorted array
  • Container with most water
  • 3Sum, 4Sum problems

Key Indicators:

  • ✓ Sorted or can be sorted
  • ✓ Need to find pairs/triplets
  • ✓ Optimization with O(n) instead of O(n²)

🪟Sliding Window

Medium O(n)

Maintain a window of elements while sliding it across the array/string to find optimal subarrays or substrings.

When to Use:

  • Maximum/minimum sum subarray of size K
  • Longest substring with K distinct characters
  • Minimum window substring
  • Anagram/permutation problems

Key Indicators:

  • ✓ Contiguous sequence (subarray/substring)
  • ✓ Find max/min/longest/shortest
  • ✓ Optimize from O(n²) to O(n)

➕Prefix Sum

Easy O(n)

Precompute cumulative sums to answer range sum queries efficiently in O(1) time.

When to Use:

  • Range sum queries
  • Subarray sum equals K
  • Contiguous array problems
  • Product of array except self

Key Indicators:

  • ✓ Multiple range queries
  • ✓ Sum of subarrays
  • ✓ Need O(1) query time

📈Kadane's Algorithm

Medium O(n)

Find maximum sum of contiguous subarray using dynamic programming approach.

When to Use:

  • Maximum subarray sum
  • Maximum product subarray
  • Best time to buy/sell stock
  • Maximum circular subarray sum

Key Indicators:

  • ✓ Maximum/minimum subarray
  • ✓ Contiguous elements
  • ✓ Can have negative numbers

🔄Cyclic Sort

Easy O(n)

Place each number at its correct index when dealing with numbers in range [1, n].

When to Use:

  • Find missing number
  • Find duplicate number
  • Find all duplicates
  • First missing positive

Key Indicators:

  • ✓ Array with numbers 1 to n
  • ✓ Find missing/duplicate
  • ✓ O(1) space required

🔀In-place Array Manipulation

Easy-Medium O(n)

Modify array in-place without using extra space, often using swap or reverse techniques.

When to Use:

  • Rotate array
  • Move zeros to end
  • Sort colors (Dutch flag)
  • Rearrange array elements

Key Indicators:

  • ✓ O(1) space constraint
  • ✓ Need to rearrange/sort
  • ✓ Can modify input

String-Specific Patterns

Patterns designed specifically for string manipulation

🎯Pattern Matching

Medium O(n)

Find occurrences of pattern in text using KMP, Rabin-Karp, or Z-algorithm.

When to Use:

  • Find pattern in string
  • Repeated substring pattern
  • Longest prefix/suffix
  • String matching problems

🪞Palindrome

Easy-Medium O(n)

Check for palindromes or find palindromic substrings using expand-around-center or DP.

When to Use:

  • Valid palindrome
  • Longest palindromic substring
  • Palindrome partitioning
  • Count palindromic substrings

📝Anagram/Permutation

Easy O(n)

Use character frequency maps to solve anagram and permutation problems.

When to Use:

  • Valid anagram
  • Group anagrams
  • Find all anagrams
  • Permutation in string

Linked List Patterns

Essential techniques for linked list manipulation

🐰🐢Fast & Slow Pointers

Easy-Medium O(n)

Use two pointers moving at different speeds to detect cycles or find middle elements.

When to Use:

  • Detect cycle in linked list
  • Find middle of linked list
  • Find cycle start
  • Happy number problem

Key Indicators:

  • ✓ Cycle detection needed
  • ✓ Find middle element
  • ✓ O(1) space required

↩️In-place Reversal

Medium O(n)

Reverse linked list or parts of it in-place using pointer manipulation.

When to Use:

  • Reverse linked list
  • Reverse nodes in k-group
  • Reverse between positions
  • Palindrome linked list

Key Indicators:

  • ✓ Need to reverse
  • ✓ O(1) space
  • ✓ In-place modification

🔗Merge Lists

Easy-Medium O(n)

Merge multiple sorted linked lists or combine lists using dummy node technique.

When to Use:

  • Merge two sorted lists
  • Merge k sorted lists
  • Add two numbers
  • Flatten multilevel list

Tree & Binary Tree Patterns

Traversal and manipulation techniques for tree structures

⬇️DFS (Depth-First Search)

Easy-Medium O(n)

Traverse tree depth-wise using recursion or stack. Three variants: preorder, inorder, postorder.

When to Use:

  • Tree traversals (pre/in/post-order)
  • Path sum problems
  • Validate BST
  • Diameter/height of tree

Key Indicators:

  • ✓ Explore all paths/nodes
  • ✓ Calculate tree properties
  • ✓ Top-down or bottom-up

➡️BFS (Level-Order Traversal)

Easy-Medium O(n)

Traverse tree level by level using a queue to explore nodes breadth-wise.

When to Use:

  • Level order traversal
  • Right/left side view
  • Zigzag level order
  • Find level of node

Key Indicators:

  • ✓ Level-by-level processing
  • ✓ Find shortest path in tree
  • ✓ Level-specific operations

🔍Binary Search Tree

Medium O(log n)

Leverage BST properties (left < root < right) for efficient search and operations.

When to Use:

  • Validate BST
  • Kth smallest element
  • Lowest common ancestor (BST)
  • Insert/delete in BST

Key Indicators:

  • ✓ Tree is BST
  • ✓ Need sorted order
  • ✓ Search in O(log n)

🎯Tree DP

Hard O(n)

Use dynamic programming on trees, typically computing values from children to parent.

When to Use:

  • House robber III
  • Binary tree cameras
  • Maximum path sum
  • Tree diameter problems

Graph Patterns

Essential algorithms for graph traversal and problem-solving

🔙Backtracking

Medium-Hard O(2ⁿ)

Explore all possible solutions by building candidates incrementally and abandoning invalid paths.

When to Use:

  • Generate permutations/combinations
  • Subset sum problems
  • N-Queens, Sudoku solver
  • Word search, path finding

Key Indicators:

  • ✓ Find ALL solutions
  • ✓ Constraint satisfaction
  • ✓ Generate combinations

🌐Graph DFS/BFS

Medium O(V+E)

Traverse graphs using DFS (stack) or BFS (queue) to explore nodes and edges.

When to Use:

  • Number of islands
  • Clone graph
  • Course schedule
  • Shortest path (BFS)

🔗Union-Find

Medium O(α(n))

Track connected components efficiently with union and find operations.

When to Use:

  • Number of provinces
  • Redundant connection
  • Accounts merge
  • Kruskal's MST algorithm

Key Indicators:

  • ✓ Connected components
  • ✓ Cycle detection
  • ✓ Dynamic connectivity

📊Topological Sort

Medium O(V+E)

Linear ordering of vertices in a directed acyclic graph (DAG).

When to Use:

  • Course schedule
  • Task scheduling
  • Build dependencies
  • Alien dictionary

🛣️Shortest Path

Medium-Hard O(E log V)

Find shortest path using Dijkstra's, Bellman-Ford, or Floyd-Warshall algorithms.

When to Use:

  • Network delay time
  • Cheapest flights
  • Path with minimum effort
  • Shortest path problems

Advanced Patterns

Specialized techniques for complex problems

🔍Binary Search

Easy-Medium O(log n)

Search in sorted space by repeatedly dividing search interval in half.

When to Use:

  • Search in sorted array
  • Find peak element
  • Search in rotated array
  • Find minimum in rotated array

Key Indicators:

  • ✓ Sorted or partially sorted
  • ✓ Search space can be halved
  • ✓ Need O(log n) solution

🎯Modified Binary Search

Medium-Hard O(log n)

Apply binary search on answer space or non-traditional sorted structures.

When to Use:

  • Capacity to ship packages
  • Koko eating bananas
  • Split array largest sum
  • Minimize max distance to gas station

📚Monotonic Stack

Medium O(n)

Maintain stack in monotonic order to find next/previous greater/smaller element.

When to Use:

  • Next greater element
  • Daily temperatures
  • Largest rectangle in histogram
  • Trapping rain water

Key Indicators:

  • ✓ Next/previous greater/smaller
  • ✓ Need O(n) solution
  • ✓ Range queries on array

🎚️Monotonic Queue

Hard O(n)

Maintain deque in monotonic order for sliding window maximum/minimum problems.

When to Use:

  • Sliding window maximum
  • Shortest subarray with sum ≥ K
  • Jump game variations
  • Constrained subsequence sum

⏱️Interval Merging

Medium O(n log n)

Sort intervals and merge overlapping ones for scheduling and range problems.

When to Use:

  • Merge intervals
  • Insert interval
  • Meeting rooms
  • Non-overlapping intervals

Key Indicators:

  • ✓ Intervals/ranges given
  • ✓ Overlapping detection
  • ✓ Scheduling problems

💰Greedy Algorithm

Medium Varies

Make locally optimal choice at each step hoping to find global optimum.

When to Use:

  • Jump game
  • Gas station
  • Task scheduler
  • Minimum number of arrows

🔢Bit Manipulation

Easy-Medium O(1) - O(n)

Use bitwise operations for efficient solutions to certain problems.

When to Use:

  • Single number
  • Number of 1 bits
  • Power of two
  • Reverse bits

Key Indicators:

  • ✓ XOR properties needed
  • ✓ Binary representation
  • ✓ Optimize with bits

⛰️Top K Elements

Medium O(n log k)

Use heap/priority queue to find K largest, smallest, or most frequent elements.

When to Use:

  • Kth largest element
  • Top K frequent elements
  • K closest points
  • Find median from data stream

🎲Reservoir Sampling

Medium O(n)

Randomly select K items from stream of unknown size with uniform probability.

When to Use:

  • Random pick from stream
  • Linked list random node
  • Sample from large dataset
  • Shuffle algorithm

Complexity Hints

Time Complexity Clues:

  • O(1): Math formula, bit manipulation
  • O(log n): Binary search, balanced tree operations
  • O(n): Single pass, two pointers, sliding window, linear scan
  • O(n log n): Sorting, heap operations, modified binary search
  • O(n²): Nested loops, DP with 2D table, checking all pairs
  • O(2ⁿ): Backtracking, generate all subsets/permutations

✅ Pattern Recognition Checklist:

  1. Read carefully: Understand input, output, and constraints
  2. Identify keywords: Look for hints like "sorted", "contiguous", "all solutions"
  3. Check constraints: Size limits hint at acceptable time complexity
  4. Consider data structure: Array, string, tree, graph dictate available patterns
  5. Think of similar problems: Have you solved something like this before?
  6. Start simple: Brute force first, then optimize with patterns

Practice Strategy

📚 How to Master Patterns:

  1. Learn One Pattern at a Time: Deeply understand one pattern before moving to next
  2. Solve 5-10 Problems per Pattern: Build muscle memory and recognition
  3. Code Templates: Create reusable templates for each pattern
  4. Mix Practice: After learning several patterns, solve random problems
  5. Time Yourself: Practice recognizing patterns quickly under time pressure
  6. Review Regularly: Revisit patterns weekly to maintain familiarity

Common Mistakes to Avoid

⚠️ Pattern Pitfalls:

  • Forcing a pattern: Not every problem fits a standard pattern
  • Overlooking constraints: Constraints often hint at the right approach
  • Ignoring edge cases: Empty input, single element, duplicates, etc.
  • Premature optimization: Get brute force working first
  • Not testing: Always test with examples before submission
  • Skipping analysis: Understand why a pattern works, not just how

🎤 Interview Preparation Strategy

Before the Interview

📅 1 Week Before

  • Review all pattern templates
  • Solve 2-3 problems per pattern
  • Practice on whiteboard/paper
  • Mock interviews with peers

📅 1 Day Before

  • Review pattern cheat sheet
  • Solve 2-3 easy problems
  • Rest well, stay confident
  • Prepare questions to ask

During the Interview

🎯 Step-by-Step Approach:

  1. Clarify (2 min): Ask about input size, edge cases, expected output
  2. Example (2 min): Walk through example, verify understanding
  3. Approach (5 min): Identify pattern, discuss brute force, then optimize
  4. Complexity (2 min): State time/space complexity before coding
  5. Code (25 min): Write clean code, explain as you go
  6. Test (5 min): Test with examples, edge cases
  7. Optimize (5 min): Discuss improvements if time permits

🚫 Common Interview Mistakes:

  • Jumping to code too quickly - Always discuss approach first
  • Staying silent - Think out loud, communicate your thought process
  • Getting stuck - Ask for hints, don't waste time in silence
  • Ignoring interviewer hints - Listen carefully to suggestions
  • Not testing - Always test your solution with examples
  • Poor variable names - Use meaningful names, not just i, j, k
  • Giving up - Show persistence and problem-solving ability

Communication Template

💬 What to Say During Interview:

1. Understanding the Problem:

"Let me make sure I understand correctly. We need to [restate problem]. The input is [describe input], and we should return [describe output]. Can I assume [ask about constraints]?"

2. Identifying the Pattern:

"This looks like a [pattern name] problem because [reason]. The typical approach would be to [describe approach]. This should give us O(n) time complexity."

3. Discussing Trade-offs:

"We could use [approach 1] which is simpler but takes O(n²) time, or we could use [approach 2] with O(n) time but O(n) extra space. Given the constraints, I think [chosen approach] is better because [reason]."

4. While Coding:

"I'm going to use [data structure] here to [purpose]. First, I'll handle the [base case], then [main logic]."

5. Testing:

"Let me test this with the example: [walk through]. I should also check edge cases like [empty input, single element, etc.]."
