🌈 DSA Roadmap: Arrays in Python

1️⃣ Definition & Intuition

Array = Collection of elements stored in contiguous memory locations.

Think of it as a row of lockers 🧳 — each locker has a number (index), and you can instantly access any locker.

Indexes → 0 1 2 3 4 Values → [10, 20, 30, 40, 50]

Key property: Random access is O(1) — directly jump to any index.

2️⃣ Common Operations & Time Complexity

OperationAverage TimeDescription
AccessO(1)arr[i]
Insert (end)O(1)append()
Insert (middle)O(n)shift elements
DeleteO(n)shift elements
SearchO(n)linear scan
# Python example
nums = [1, 2, 3, 4, 5]
nums.append(6)        # Add at end
nums.insert(2, 10)    # Insert at index 2
nums.remove(4)        # Remove value 4
print(nums[3])        # Access index 3

3️⃣ ASCII Visual: Array in Memory

Memory Address (Contiguous)

1000  ─▶ [  5 ]  ← arr[0]
1004  ─▶ [ 12 ]  ← arr[1]
1008  ─▶ [ 25 ]  ← arr[2]
1012  ─▶ [ 38 ]  ← arr[3]

Each element has equal spacing (fixed size). That’s why array access is O(1).

4️⃣ Problem-Solving Mindset

When facing an array problem, ask yourself 👇

  • Do I need to traverse it? (→ use loops or two-pointers)
  • Do I need to search? (→ linear or binary search)
  • Do I need to modify? (→ insertion/deletion, watch for O(n))
  • Do I need prefix/suffix results? (→ precompute partial sums)
  • Do I need window/sliding window? (→ continuous subarray)

🎯 Always think: What pattern fits this?

Array Problem Patterns:

1. Traversal → for/while loop
2. Two Pointers → sorted arrays, pairs, duplicates
3. Sliding Window → subarray, sum, longest substring
4. Prefix Sum → cumulative results
5. Hashing → frequencies, complements
6. Sorting + Binary Search → optimization

# Find max subarray sum (Kadane's Algorithm)
def maxSubArray(nums):
    max_sum = curr = nums[0]
    for x in nums[1:]:
        curr = max(x, curr + x)
        max_sum = max(max_sum, curr)
    return max_sum

# Two Pointers (Remove Duplicates)
def removeDuplicates(nums):
    if not nums: return 0
    slow = 0
    for fast in range(1, len(nums)):
        if nums[fast] != nums[slow]:
            slow += 1
            nums[slow] = nums[fast]
    return slow + 1

7️⃣ How to Think Like an Array Engineer 🧠

  • Draw array indexes on paper.
  • Trace loops manually for small input.
  • Recognize repetitive patterns → replace with formula or prefix sums.
  • Notice when a window “moves” → apply sliding window.
  • When you re-use results → caching, prefix, or DP.

Key Memory Tip: Arrays are like railway coaches 🚆 — fixed sequence, constant spacing. You can’t insert a new coach in the middle without shifting all others.
